partie2:multi-platform interactive hashtable visualizer

1. Le Travail Demandé (Étape par Étape)
Le but final est de créer un Visualiseur interactif de Table de Hachage. Vous allez construire le cœur de l'application une seule fois, puis changer son "habillage" (l'interface) à chaque étape :
Étape 1 : L'application Console (Framework Nuts)
Créer la structure de la table de hachage (XHashtable).Créer un affichage textuel mais coloré et aligné dans le terminal pour voir les cases vides, occupées ou supprimées.Créer une invite de commande interactive (CLI) pour que l'utilisateur puisse taper add 10, delete 5, find 10.
Étape 2 : L'interface Graphique Java SwingTransformer la console en une vraie fenêtre avec des boutons à la place des commandes textuelles.Dessiner manuellement la table de hachage sous forme de cases/rectangles graphiques.Ajouter des animations visuelles (clignotements, changements de couleurs) pour voir le cheminement de l'algorithme (surtout lors d'une collision).Étape 3 : La transition vers JavaFXRecréer l'interface avec une technologie plus moderne (JavaFX) en utilisant un Canvas ou des composants graphiques évolués, d'abord intégrés (JavaFX 8), puis sous forme modulaire autonome (JavaFX 17+ avec Scene Builder).Étape 4 : L'application Web (JavaScript Canvas)Réimplémenter toute l'interface visuelle en utilisant le langage du Web (HTML/JavaScript) et la balise <canvas> pour le dessin.2. Les Notions Clés à Comprendre (Votre Bagage Théorique)Pour mener à bien ce projet, vous devez être à l'aise avec ces 4 grands piliers :A. L'Algorithmique : La Table de Hachage & Les CollisionsFonction de hachage : Comprendre comment transformer une valeur (ex: 34) en un indice de tableau fixe (ex: 4) grâce à l'opérateur modulo (%).La Collision : Savoir que deux clés différentes peuvent donner le même indice.La Résolution par Sondage Linéaire : Comprendre que si la case $h(k)$ est occupée, l'algorithme va chercher la case suivante ($h(k)+1$, puis $h(k)+2$...) jusqu'à trouver une place libre.L'état des cases : Distinguer une case Vide (jamais touchée), une case Occupée (contient une valeur), et une case Supprimée (valeur spéciale pour ne pas casser la recherche des éléments suivants).B. L'Architecture Logicielle : Le Pattern MVCModèle (Model) : Votre classe XHashtable. Elle stocke les données brutes et applique les formules logiques. Elle ne contient aucun code graphique ou d'affichage.Vue (View) : La classe chargée de dessiner (soit avec NOut dans la console, soit avec Graphics2D dans Swing). Elle lit le Modèle et le traduit visuellement.Contrôleur (Controller) : La classe qui intercepte les actions (une commande tapée au clavier ou le clic sur un bouton), demande au Modèle de se mettre à jour, puis demande à la Vue de se rafraîchir.C. Le Framework d'Affichage Console : NutsFormatage Rich-Text : Comprendre l'utilisation des balises comme ##:green,bold:texte## pour colorer la console.Alignement fixe : Comprendre la syntaxe ##:5:%s## qui force chaque case de votre table à mesurer exactement 5 caractères de large pour que l'affichage reste parfaitement droit et lisible.D. Le Graphisme Java (Pour l'Étape 2)Le composant JPanel : Comprendre que c'est une zone de dessin vierge.La méthode paintComponent(Graphics2D g) : Comprendre que c'est le "moteur de rendu" que Java appelle pour dessiner des formes géométriques (drawRect, fillRect, drawString) avec un pinceau virtuel g.Le rafraîchissement (repaint()) : Comprendre comment demander à l'interface de se redessiner dès que le modèle change ou pour créer une animation.
